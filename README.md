# ระบบคัดกรองต้อกระจกจากภาพถ่ายตา (เว็บ รันในเบราว์เซอร์)

เว็บคัดกรอง **ต้อกระจกระยะสุก** จากภาพดวงตาที่ถ่ายด้วยมือถือ ทำงาน **ในเบราว์เซอร์ทั้งหมด**
ด้วย [ONNX Runtime Web](https://onnxruntime.ai/docs/tutorials/web/) — **ไม่ต้องมีเซิร์ฟเวอร์ ไม่ต้องรัน Python**
รูปภาพไม่ถูกส่งออกไปไหน ประมวลผลบนเครื่องผู้ใช้ล้วนๆ

> ⚠️ เป็น **เครื่องมือคัดกรองเบื้องต้น ไม่ใช่การวินิจฉัย** — ผลผิดปกติต้องพบจักษุแพทย์เสมอ

## ทำงานยังไง (pipeline 2 ชั้น)

1. **ประตู (gate)** — MobileNetV3 ตรวจว่าภาพนี้ "เป็นตา" ไหม ถ้าไม่ใช่ → ตอบ *"ไม่ใช่รูปตา"*
2. **cataract** — ถ้าเป็นตา → ensemble (EfficientNet-B0 + MobileNetV3) + TTA → *ปกติ / ต้อกระจก*

โมเดลอยู่ในโฟลเดอร์ `models/` (ONNX, ใส่ sigmoid ในตัวแล้ว) โหลดครั้งแรกครั้งเดียว (~27 MB)

## เปิดใช้ออนไลน์ด้วย GitHub Pages (ฟรี)

1. push โค้ดนี้ขึ้น GitHub repo
2. ไปที่ **Settings → Pages**
3. **Source:** เลือก `Deploy from a branch` → Branch `main` → โฟลเดอร์ `/ (root)` → Save
4. รอสักครู่ เว็บจะขึ้นที่ `https://<username>.github.io/<repo>/`

เปิดลิงก์นั้นในมือถือ/คอมได้เลย กล้องถ่ายสดใช้ได้เพราะ GitHub Pages เป็น https

## รันในเครื่อง (ทดสอบ)

ต้องเปิดผ่าน http (เปิดไฟล์ตรง ๆ จะโหลดโมเดลไม่ได้เพราะ CORS):

```bash
python -m http.server 8080
# แล้วเปิด http://localhost:8080
```

## รายละเอียดเทคนิค

- อินพุต 224×224, normalize ด้วย ImageNet mean/std, TTA = ภาพปกติ + พลิกซ้ายขวา
- threshold: ประตู `prob_eye ≥ 0.50`, ต้อกระจก `prob ≥ 0.25` (เน้น sensitivity สำหรับงานคัดกรอง)
- ผลบน gold test: ประตูปฏิเสธ non-eye 100%, cataract AUC 0.992 / sensitivity 0.98 / specificity 0.98
- inference ในเบราว์เซอร์ให้ผล **ตรงกับ backend Python ทุกค่า**

## ข้อจำกัด

- ตรวจได้เฉพาะ **ระยะสุก** ที่เห็นรูม่านตาขาว — ระยะเริ่มต้นตรวจจากภาพภายนอกไม่ได้
- ยังไม่แยก retinoblastoma / corneal opacity / Coats disease — เด็กที่เห็นรูม่านตาขาวต้องพบแพทย์ด่วน
- โมเดลเทรนจากภาพเว็บ ไม่มี clinical grading — เหมาะเป็น prototype/งานเรียน
